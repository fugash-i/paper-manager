# 要件定義

## なにがしたいか

PCの買い替えに伴って、以前使用していたpapntをベースとした論文管理ツールを作成する. 

## 機能要件

| 項目 | 内容 |
| --- | --- |
| 入力ソース | DOI, ローカルのPDFファイル, またはURLから情報を取得できること.  |
| メタデータ取得 | 論文に結びついたメタデータを自動取得する.  |
| Notion連携 | Notionのデータベースに新規ページを作成し, 各プロパティを埋める.  |
| テンプレート生成 | 指定した4項目（abstract, introduction, Result, Discussion）のテンプレを本文に作成する.  |
| 翻訳サービスとの連携 | abstractの和訳を書き込んでおく.  |

## 非機能要件

- 動作環境: macOS(apple M4 tip)で機能する.
- pythonパッケージは基本condaで管理する.
- ソースコードはモジュール化する.

### 1. main.py

実行フローの制御をする. 

### 2. input_handler.py

入力を受け取り, PDFならDOIを抽出, それが正規表現かどうか確認する. 

- **正規表現チェック:** 入力が `10.xxxx/xxxx` の形式ならDOIとして扱う。
- **ファイル存在確認:** ローカルパスなら、それがPDFファイルか確認し、PDFからDOIを抽出（`PyMuPDF`等を使用）する。
- **エラー処理:** 正しくない入力に対し、「有効なDOIではありません」といった警告を出す。

### 3. metadata_fetcher.py

外部データベースにアクセスし, 書誌情報をとってくる. Crossref, PubMed, arXiv 等のAPIを叩いてデータを整形。

- API連携: DOIをキーに情報をリクエスト。
- タイトル整合性チェック: 以前のルール通り、取得したメタデータのタイトルと実際の論文が一致するか再確認。
- データ整形: 著者リストを「第一著者 et al.」形式にするなどの整形処理。

### 4. translator.py

外部翻訳APIとの通信: これは後で詳細を詰める. 

### 5. notion_api.py

Notion APIを用いて, 実際にページを作成する. 

- Propertyの作成: タイトル、DOI、URL、出版年などをデータベースの列に流し込む。
- Blockの構築: 本文（Page Content）に、`# Abstract` や `# Introduction` といった見出し（Heading）と、翻訳結果を階層構造で書き込む。

### 6. config.py / .env

APIキーやnotion DB IDなどを管理. githubにはuploadしない. 

# 具体的な実装

## translator

- Gemini APIを使った翻訳を行う.

## notion databaseのデータベース設計

- 論文タイトル
- First Author
- 出版年
- PDF(添付用の空のカラムを作成)
- ジャーナル
- パブリッシャー
- 既読/未読のチェックボックス
- Cite key: Cite keyの特定のタグをつけたものだけを参考文献として出力(bib形式, Natureの参考文献形式のテキストファイル)できるようにする
- DOI
- DOIのリンク
- PMID
